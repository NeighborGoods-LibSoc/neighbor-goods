# Skill: Philosophical Alignment (Library Socialism)

This skill ensures that all code, content, and features align with the core principles of NeighborGoods and Library Socialism.

## Core Concepts
- **Usufruct**: Use-rights based on need. Design for sharing, not exclusive ownership.
- **Irreducible Minimum**: Ensure basic needs are met without conditions.
- **Non-Extractive**: No ads, data harvesting, or rent-seeking.
- **Complementarity**: Generate value through non-hierarchical community interaction.

## Guidelines
1. **Terminology**: Use "neighbors," "sharing," "solidarity," and "mutual aid" instead of "users," "customers," "market," or "charity."
2. **Feature Design**: When proposing features, ask: "Does this increase community trust or just technical efficiency?"
3. **Accessibility**: Prioritize simple, inclusive interfaces for all technical literacy levels.
4. **Decentralization**: Always design for a federated model (many small, connected nodes) rather than a single central platform.
